# 工单系统使用手册（流程版）

## 适用范围
本手册用于指导“从下单到财务结算”的完整业务流程，结合后端逻辑与前端功能现状梳理关键步骤、角色职责、页面入口、以及当前仍需补齐的功能。

## 角色与职责
- 业务员/销售
  - 新建客户订单、维护客户与产品信息
  - 提交客户订单、跟进审核结果
  - 更新付款信息、回访客户交付
- 审核人/主管
  - 审核客户订单/施工单，必要时退回或驳回
  - 对紧急订单进行标记与审批
- 计划/制表人
  - 新建施工单，配置工序、交期、优先级
  - 维护与客户订单的关联关系
- 生产主管
  - 任务分派、人员调度、异常跟进
  - 监控任务完成率与工序进度
- 操作员
  - 认领任务、更新进度、提交完成
  - 记录不良品与备注
- 仓库/物流
  - 入库、出库、发货单管理
  - 记录签收/出库原因
- 质检
  - 质检检验、记录结果
- 财务
  - 录入发票、收付款、对账单

## 全流程总览（下单 -> 结算）
1. 客户订单
2. 审核/批准
3. 生成施工单
4. 工序拆分与任务分派
5. 生产执行/进度更新
6. 入库/出库/发货
7. 付款与结算

下文按步骤展开说明。

## 1. 客户订单（下单）
**入口**：`客户订单` 列表与详情页

**操作流程**
1. 业务员新建客户订单（客户、交期、产品明细、数量等）。
2. 订单初始状态：`draft`（草稿）。
3. 业务员提交订单：状态变为 `submitted`（已提交）。
4. 审核人审核：通过 -> `approved`；拒绝 -> `rejected`。
5. 审核通过后可“开始生产”，状态进入 `in_production`。
6. 客户订单完成或取消：`completed`/`cancelled`。

**关键操作**
- 提交/审核/拒绝/开始生产/完成/取消
- 更新付款（已付金额/付款时间）

## 2. 审核与审批
**入口**：`审批中心`、`审批工作流`、`审批步骤`、`紧急订单`

**说明**
- 系统支持普通审批与多级审批。
- 施工单支持“重新审核申请”流程。

## 3. 生成施工单
**入口**：`施工单` 列表/详情

**来源**
- 由客户订单“生成施工单”。
- 也可直接创建施工单（视权限）。

**施工单关键字段**
- 客户、交期、优先级
- 工序列表（顺序、部门、时间计划）
- 状态：`pending/in_progress/paused/completed/cancelled`
- 审核状态：`pending/approved/rejected`

**说明**
- 施工单审批通过后进入生产阶段。
- 修改工序后可触发“任务同步预览/执行”（后端已支持）。

## 4. 工序拆分与任务分派
**入口**：
- `任务列表`
- `部门任务看板`
- `主管看板`
- `操作员任务中心`

**任务状态**
- `draft`、`pending`、`in_progress`、`completed`、`cancelled`

**分派方式**
- 主管或管理人员将任务分派给部门或操作员
- 操作员可在“任务中心”认领任务

**协同功能**
- 任务拆分（多人协作）
- 任务统计与分派历史

## 5. 生产执行与进度更新
**入口**：`任务列表`、`部门任务看板`、`操作员任务中心`

**操作员流程**
1. 查看“我的任务”或可认领任务
2. 认领后更新进度（完成数量、不良数量、备注）
3. 提交完成

**主管流程**
- 在看板/列表中更新任务进度
- 拖拽切换任务状态（看板）

## 6. 入库 / 出库 / 发货
**入口**
- 入库：`入库管理`
- 出库：`出库管理`
- 发货：`发货管理`

**说明**
- 生产完成后根据任务完成数量入库
- 出库通常由客户订单驱动
- 发货完成可触发客户订单完成状态

## 7. 财务结算
**入口**
- `发票管理`
- `收付款管理`
- `对账单`

**说明**
- 客户订单支持付款状态维护
- 财务模块用于票据、收付款与对账流程

## 如何查看“自己的任务和完成情况”
- 进入 `操作员任务中心`
  - 查看“我的任务”与“可认领任务”
  - 更新进度与完成状态
- 进入 `任务列表`
  - 使用筛选与搜索查看个人任务
- 进入 `通知中心`
  - 实时获取任务分派与变更通知

- 客户订单创建/审核/状态流转

## 仍需完善（代码层缺口清单）
1. **附件/文件上传体系**
   - 施工单设计文件（`design_file`）缺少前端上传与预览。
   - 质检报告附件、发票附件、签收照片等后端字段已存在，但前端无上传入口。

2. **真正的甘特/计划排程**
   - 当前已提供“时间轴视图”，但未基于工序计划开始/结束时间绘制甘特。
   - 后端已有工序计划时间字段（`planned_start_time/planned_end_time`），可扩展。

3. **任务同步预览/执行 UI**
   - 后端支持工序变更后的任务同步预览/执行接口，前端尚未提供入口。

4. **统计分析可视化不足**
   - 工作台已显示基础指标，但未完整呈现：任务统计、效率分析、客户/产品分布等图表。

5. **交付/出库联动提示**
   - 后端对客户订单与出库/交付联动已有逻辑，但前端缺少更明确的状态提示与流程提醒。

## 建议的下一步完善顺序
1. 附件/文件上传能力（覆盖设计文件、质检报告、发票、签收）
2. 甘特排程视图（基于工序计划时间）
3. 任务同步预览/执行 UI
4. 仪表盘与统计图表完善
